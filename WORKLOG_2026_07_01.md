# Worklog — 2026-07-01

## Session Goal
Fix the 404/500 error on the Opportunity Detail page in the Sales Pipeline.

---

## Root Cause
The `OpportunityDetail` table in SQL Server was missing the `nextFollowUpDate` column that had been added to `prisma/schema.prisma`. Every `prisma.deal.findFirst()` call that included `opportunityDetail: true` threw a Prisma `P2022` error (`Invalid column name 'nextFollowUpDate'`), which surfaced as:

- **500** on `GET /api/opportunities/<id>`
- The frontend rendered the generic **"Opportunity not found"** fallback (appearing as a 404 to the user)

The dev server log confirmed the exact error:
```
prisma:error Invalid column name 'nextFollowUpDate'.
[GET /api/opportunities/[id]] error: PrismaClientKnownRequestError
  code: 'P2022', meta: { modelName: 'Deal', column: 'nextFollowUpDate' }
```

---

## Changes Made

### 1. `app/api/opportunities/[id]/route.ts` — Error handling improvement
- Wrapped the entire `GET` handler body in a `try/catch` block.
- On any unhandled error, logs to console with `[GET /api/opportunities/[id]]` prefix and returns a proper **500** JSON response with the error message.
- **Before:** Unhandled Prisma errors caused an opaque 500 with no useful message, which the frontend interpreted as "not found".
- **After:** Errors are logged and surfaced clearly, making future debugging straightforward.

### 2. Database schema sync
- Ran `npx prisma db push --accept-data-loss` to sync the SQL Server database with `prisma/schema.prisma`.
- This added the missing `nextFollowUpDate` column to the `OpportunityDetail` table.
- **Side effect:** The sync also dropped schema objects that existed in DB but were removed from the Prisma schema:
  - `sourceMeta` column on `Lead` table (1 non-null value lost)
  - `WebhookLog` table (6 rows lost)
- Verified column existence via `INFORMATION_SCHEMA.COLUMNS` query — confirmed present.

### 3. `next.config.ts` — Legacy URL redirects (already in working copy)
- Added redirect rules for underscore/legacy URL variants:
  - `/sales_pipeline/:id/opportunity_detail` → `/sales-pipeline/:id/opportunity-detail`
  - `/sales_pipeline/:path*` → `/sales-pipeline/:path*`

### 4. `prisma/schema.prisma` — New field (already in working copy)
- Added `nextFollowUpDate DateTime?` to `OpportunityDetail` model (line 547).

### 5. `app/(dashboard)/sales-pipeline/[id]/opportunity-detail/page.tsx` — UI improvements (already in working copy)
