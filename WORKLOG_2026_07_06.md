## 2. Negotiation ↔ Quotation Revision Fix

**Problem:** Price revisions in negotiation created unrelated quotation records with no link back; approving/rejecting a quotation never updated the negotiation; the revision author could approve their own revision; no escalation routing.

### Root causes found (investigation)
- `Quotation.revisionNumber` existed but no `negotiationId` FK or `parentQuotationId` grouping
- `PUT /api/quotations/[id]/approval` never touched the negotiation record
- No self-approval check; approver was always SalesManager regardless of discount size
- Clone route created fully independent quotations

### Schema changes (`prisma/schema.prisma`)
- `Quotation.negotiationId` — FK to Negotiation (`NegotiationQuotationRevisions` relation)
- `Quotation.parentQuotationId` — self-relation grouping revisions under a root quotation
- `QuotationApproval.requiredApproverRole` — escalation role captured at creation time
- `QuotationApproval.revisionAuthorId` — tracks revision author for self-approval prevention
- Back-relations added on `Negotiation.quotationRevisions` and `User.quotationRevisionAuthors`

### Behavior changes
- **`app/api/negotiations/[id]/revisions/route.ts`** — creating a price revision now auto-clones the linked quotation as a numbered revision (R2, R3, …) with `negotiationId` + `parentQuotationId` set, snapshots the prior revision, and creates a `QuotationApproval` with the correct approver role when approval is required
- **`app/api/quotations/[id]/approval/route.ts`** —
  - Self-approval blocked: revision author cannot approve their own revision (403 with clear message)
  - Escalation role enforced: approver's role must match `requiredApproverRole` (Admin bypasses)
  - Quotation status now set to Approved/Rejected
  - Linked negotiation synced in the same transaction: **Approved → `PriceRevision`** (revision marked Approved, `discountApproved` + `approvedById` set); **Rejected → `Active`** (revision marked Rejected, negotiator can revise again)
- **`app/api/quotations/[id]/request-approval/route.ts`** — escalation routing: when discount exceeds threshold, approval goes to the escalation role instead of SalesManager
- **`app/api/quotations/[id]/clone/route.ts`** — accepts optional `negotiationId`; sets `negotiationId` + `parentQuotationId` on the clone

### Escalation config (reuses existing `SystemConfig` key-value pattern)
| Key | Default |
|-----|---------|
| `approval_matrix_escalation_threshold` | 15 (%) |
| `approval_matrix_escalation_role` | `SalesDirector` |

### Frontend
- Quotation list: `R{n}` badge for revisions > 1, "NEG" link to linked negotiation, `PendingApproval`/`Approved` status styles
- Quotation detail: "Negotiation" link badge in header, approval banner shows required approver role + self-authorship warning, status refreshes after approve/reject via existing `loadQuotation()`
- Detail API includes `revisionAuthor` in approval data
