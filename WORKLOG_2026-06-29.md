# Work Log - June 29, 2026

## Product Catalogue Module Implementation

### Overview
Completed full implementation of the Product Catalogue Module for Suki CRM, including database schema updates, backend API routes, seed data, UI pages, variant gating, and integration components.

### Schema Updates
- **File:** `prisma/schema.prisma`
- **Changes:**
  - Added `ProductDatasheet` model with fields: id, companyId, productId, title, fileName, fileUrl, isActive, createdAt, updatedAt
  - Added `ProductBrochure` model with fields: id, companyId, productId, title, fileName, fileUrl, isActive, createdAt, updatedAt
  - Updated `Product` model to include relations to `ProductDatasheet` and `ProductBrochure`
  - Removed legacy `datasheetUrl` and `brochureUrl` string fields from `Product` model
  - Changed `Product.productCode` unique constraint from global to tenant-scoped: `@@unique([companyId, productCode])`
- **Migration:** Used `npx prisma db push` due to SQL Server permission limitations

### API Routes
- **Base Path:** `/api/catalogue/`
- **Updated Routes:**
  - `products/[id]/datasheets/route.ts` - Refactored to use `ProductDatasheet` model (GET, POST)
  - `products/[id]/brochures/route.ts` - Refactored to use `ProductBrochure` model (GET, POST)
  - `products/[id]/route.ts` - Updated GET to include specifications, datasheets, brochures relations
- **New Routes:**
  - `products/[id]/datasheets/[dsId]/route.ts` - DELETE for individual datasheets
  - `products/[id]/brochures/[brId]/route.ts` - DELETE for individual brochures
- **Existing Routes:** Categories, Products, Specifications endpoints already functional

## Technical Notes

### Database
- **Type:** SQL Server
- **Migration Method:** `prisma db push` (due to limited CREATE DATABASE permission)
- **Prisma Version:** v6.4.1

### Tenant Isolation
- All product catalogue data scoped by `companyId`
- Product codes can be reused across tenants (unique constraint on `[companyId, productCode]`)
- API routes enforce tenant isolation via session `companyId`

### Data Model
- Categories support hierarchy (parentCategoryId)
- Products can have multiple specifications, datasheets, and brochures
- Soft delete support on Product model (deletedAt, deletedById)
